# 提示词工程与模板管理：驾驭大模型的艺术

## 什么是提示词工程？

提示词（Prompt）是与大模型交互的接口，也是指挥模型行为的"指令"。提示词工程（Prompt Engineering）就是设计和优化提示词，让模型输出更准确、更符合预期的技术。

在Agent系统中，提示词尤为重要——它不仅决定模型的回答质量，还直接影响模型的工具调用决策。

## 项目中的三套提示词

本项目根据不同场景设计了三套提示词：

### 1. 主提示词（main_prompt.txt）—— 智能客服场景

这是最核心的提示词，定义了ReAct智能体的完整行为规范：

```
你是扫地机器人和扫拖一体机器人的专业智能客服，具备自主的ReAct思考与工具调用能力，
严格遵循「思考→行动→观察→再思考」的流程回答用户问题...

### 核心思考准则
1. 先判断用户的核心需求，分析信息是否足够直接回答
2. 调用工具获取结果后，再次判断信息是否完整
3. 工具调用的入参必须与工具定义完全一致
4. 【报告生成强约束】若判断为报告生成需求，需严格遵循固定执行流程

### 可使用工具及能力边界
1. rag_summarize：从向量库检索专业资料
2. get_weather：获取城市天气信息
3. get_user_location：获取用户所在城市
4. get_user_id：获取用户ID
5. get_current_month：获取当前月份
6. fetch_external_data：获取用户使用记录
7. fill_context_for_report：触发报告上下文注入

### 输出规则
1. 每次调用工具前，必须输出真实的自然语言思考过程
2. 思考过程完成后，直接触发工具调用
3. 仅当信息足够专业、完整时，才生成最终回答
```

**设计要点**：
- 明确角色定位（专业智能客服）
- 定义思考流程（ReAct循环）
- 逐一说明工具的能力边界和使用规则
- 设置强约束（报告生成的固定流程）

### 2. RAG总结提示词（rag_summarize.txt）—— 知识检索场景

```
你是专注于"基于参考资料总结"的AI助手，需结合用户提问和向量检索到的参考资料，
生成简洁准确的概括回答。

### 输入信息
1. 用户提问：{input}
2. 参考资料：{context}

### 严格遵守以下约束
1. 事实准确：回答必须完全基于参考资料中的信息，不编造
2. 语言要求：仅用中文回答，语气客观、简洁
3. 聚焦提问：严格围绕用户原始提问总结
4. 格式要求：仅输出概括内容本身，纯文本字符串
```

**设计要点**：
- 使用`{input}`和`{context}`模板变量
- 强调"不编造"，防止幻觉
- 限定输出格式为纯文本

### 3. 报告生成提示词（report_prompt.txt）—— 报告写作场景

```
你是专业的扫地/扫拖机器人方面的报告写手。根据查询回来的信息，
写一份机器使用情况报告并给出建议。

查询信息可以使用如下工具：
1. get_user_id
2. get_current_month
3. fetch_external_data
4. rag_summarize

输出要求:
1. 使用中文生成报告，遵循MarkDown语法
2. 根据使用情况给出具体建议
3. 标题为黑马程序员扫地机器人使用情况报告与保养建议
```

**设计要点**：
- 角色从"客服"切换为"报告写手"
- 工具集缩减为报告相关的4个
- 明确输出格式（Markdown）和标题

## 提示词模板与变量

RAG提示词使用了LangChain的`PromptTemplate`：

```python
from langchain_core.prompts import PromptTemplate

prompt_text = load_rag_prompts()  # 包含{input}和{context}变量
prompt_template = PromptTemplate.from_template(prompt_text)
```

调用时传入变量值：

```python
chain.invoke({
    "input": query,       # 用户提问
    "context": context,   # 检索到的参考资料
})
```

## 提示词加载管理

```python
from utils.config_handler import prompts_conf
from utils.path_tool import get_abs_path

def load_system_prompts():
    system_prompt_path = get_abs_path(prompts_conf["main_prompt_path"])
    return open(system_prompt_path, "r", encoding="utf-8").read()

def load_rag_prompts():
    rag_prompt_path = get_abs_path(prompts_conf["rag_summarize_prompt_path"])
    return open(rag_prompt_path, "r", encoding="utf-8").read()

def load_report_prompts():
    report_prompt_path = get_abs_path(prompts_conf["report_prompt_path"])
    return open(report_prompt_path, "r", encoding="utf-8").read()
```

提示词路径通过YAML配置管理，内容存储在独立的txt文件中，实现了**配置与内容分离**。

## 提示词切换机制

提示词的动态切换通过中间件实现：

```python
@dynamic_prompt
def report_prompt_switch(request: ModelRequest):
    is_report = request.runtime.context.get("report", False)
    if is_report:
        return load_report_prompts()   # 报告场景
    return load_system_prompts()        # 客服场景
```

## 提示词设计原则

| 原则 | 说明 | 项目中的体现 |
|------|------|-------------|
| 角色明确 | 告诉模型"你是谁" | "专业智能客服"/"报告写手" |
| 流程清晰 | 定义思考步骤 | ReAct思考准则 |
| 约束具体 | 明确什么能做什么不能做 | "不编造"/"5次调用后回复我不知道" |
| 工具说明 | 详细描述每个工具 | 能力边界+使用场景+调用规则 |
| 输出规范 | 指定输出格式 | "纯文本字符串"/"Markdown格式" |

## 总结

提示词工程是AI应用开发中最具创造性的环节。好的提示词能让模型行为可预测、输出高质量。本项目通过三套场景化提示词、模板变量机制和动态切换中间件，构建了一个灵活且强大的提示词管理体系，是整个智能体系统"大脑"的核心。
