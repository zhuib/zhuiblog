# 工厂模式创建大模型实例：灵活切换模型的后端架构

## 为什么需要工厂模式？

在AI应用开发中，我们经常需要使用不同类型的模型：
- **对话模型（Chat Model）**：用于理解和生成自然语言
- **嵌入模型（Embedding Model）**：用于将文本转为向量

而且同类型模型可能有多个供应商（OpenAI、Ollama、智谱等），如果直接在代码中硬编码模型实例，切换模型时就需要修改多处代码。工厂模式将模型创建逻辑封装起来，实现**一处修改，全局生效**。

## 项目中的实现

### 抽象基类

```python
from abc import ABC, abstractmethod
from typing import Optional
from langchain_core.embeddings import Embeddings
from langchain_core.language_models.chat_models import BaseChatModel


class BaseModelFactory(ABC):
    @abstractmethod
    def generator(self) -> Optional[Embeddings | BaseChatModel]:
        pass
```

- 使用`ABC`（Abstract Base Class）定义抽象基类
- `@abstractmethod`标记抽象方法，子类必须实现
- 返回类型使用联合类型`Embeddings | BaseChatModel`

### 对话模型工厂

```python
from langchain_ollama import ChatOllama

class ChatModelFactory(BaseModelFactory):
    def generator(self) -> Optional[Embeddings | BaseChatModel]:
        return ChatOllama(model=rag_conf["chat_model_name"])
```

### 嵌入模型工厂

```python
from langchain_ollama import OllamaEmbeddings

class EmbeddingsFactory(BaseModelFactory):
    def generator(self) -> Optional[Embeddings | BaseChatModel]:
        return OllamaEmbeddings(model=rag_conf["embedding_model_name"])
```

### 全局实例

```python
chat_model = ChatModelFactory().generator()
embed_model = EmbeddingsFactory().generator()
```

通过模块级别的变量，整个项目共享同一个模型实例，避免重复创建。

## 模型配置

模型名称通过YAML配置文件管理：

```yaml
# config/rag.yml
chat_model_name: llama3.1:8b
embedding_model_name: nomic-embed-text:latest
```

切换模型只需修改配置文件，无需改动任何代码。

## 工厂模式的类图

```
BaseModelFactory (抽象基类)
    │
    ├── ChatModelFactory
    │       └── generator() → ChatOllama
    │
    └── EmbeddingsFactory
            └── generator() → OllamaEmbeddings
```

## 模型在项目中的使用位置

| 模型 | 使用位置 | 用途 |
|------|---------|------|
| chat_model | ReactAgent | 智能体推理与决策 |
| chat_model | RagSummarizeService | RAG总结生成 |
| embed_model | VectorStoreService | 文档向量化与检索 |

## 扩展：切换到其他模型供应商

如果需要从Ollama切换到OpenAI，只需修改工厂类：

```python
# 修改前
from langchain_ollama import ChatOllama

class ChatModelFactory(BaseModelFactory):
    def generator(self):
        return ChatOllama(model=rag_conf["chat_model_name"])

# 修改后
from langchain_openai import ChatOpenAI

class ChatModelFactory(BaseModelFactory):
    def generator(self):
        return ChatOpenAI(model=rag_conf["chat_model_name"])
```

所有使用`chat_model`的地方自动切换，无需逐个修改。

## Ollama本地部署

本项目使用Ollama部署本地大模型，优势包括：

- **数据隐私**：模型运行在本地，数据不出服务器
- **无API费用**：不按token计费
- **低延迟**：无需网络请求
- **离线可用**：不依赖外部API

常用命令：
```bash
# 拉取模型
ollama pull llama3.1:8b
ollama pull nomic-embed-text

# 查看已安装模型
ollama list

# 运行模型
ollama run llama3.1:8b
```

## 总结

工厂模式是AI应用中管理模型实例的最佳实践。通过抽象基类和具体工厂的分离，实现了模型创建逻辑的统一管理和灵活切换。结合YAML配置文件，切换模型只需修改一处配置，大大提高了代码的可维护性和可扩展性。
