# 日志系统设计：为智能体装上"黑匣子"

## 为什么需要日志？

在AI智能体系统中，日志是不可或缺的基础设施：

- **调试排错**：Agent的决策过程复杂，没有日志很难定位问题
- **行为审计**：记录每次工具调用和模型推理，便于回溯分析
- **性能监控**：追踪工具调用耗时、模型响应时间等
- **运行保障**：生产环境中日志是排查问题的唯一依据

## 项目中的日志实现

### 日志配置

```python
import logging
from utils.path_tool import get_abs_path
import os
from datetime import datetime

# 日志保存的根目录
LOG_ROOT = get_abs_path("logs")

# 确保日志目录存在
os.makedirs(LOG_ROOT, exist_ok=True)

# 日志格式
DEFAULT_LOG_FORMAT = logging.Formatter(
    '%(asctime)s - %(name)s - %(levelname)s - %(filename)s:%(lineno)d - %(message)s'
)
```

### 日志器创建

```python
def get_logger(
        name: str = "agent",
        console_level: int = logging.INFO,
        file_level: int = logging.DEBUG,
        log_file = None,
) -> logging.Logger:
    logger = logging.getLogger(name)
    logger.setLevel(logging.DEBUG)

    # 避免重复添加Handler
    if logger.handlers:
        return logger

    # 控制台Handler
    console_handler = logging.StreamHandler()
    console_handler.setLevel(console_level)
    console_handler.setFormatter(DEFAULT_LOG_FORMAT)
    logger.addHandler(console_handler)

    # 文件Handler
    if not log_file:
        log_file = os.path.join(LOG_ROOT, f"{name}_{datetime.now().strftime('%Y%m%d')}.log")

    file_handler = logging.FileHandler(log_file, encoding='utf-8')
    file_handler.setLevel(file_level)
    file_handler.setFormatter(DEFAULT_LOG_FORMAT)
    logger.addHandler(file_handler)

    return logger
```

### 全局日志器

```python
logger = get_logger()
```

其他模块通过导入使用：

```python
from utils.logger_handler import logger
```

## 双通道输出设计

项目采用**控制台 + 文件**双通道输出：

| 通道 | 级别 | 用途 |
|------|------|------|
| 控制台 | INFO | 开发时实时查看关键信息 |
| 文件 | DEBUG | 记录所有详细信息，用于事后分析 |

### 日志级别

```
DEBUG → INFO → WARNING → ERROR → CRITICAL
  ↑                                    ↑
最详细                              最严重
```

- **DEBUG**：详细的调试信息，如消息内容、工具参数
- **INFO**：关键流程信息，如工具调用开始/成功
- **WARNING**：非致命问题，如数据未检索到
- **ERROR**：错误信息，如文件加载失败

## 日志格式

```
2025-06-02 10:30:15,123 - agent - INFO - agent_tools.py:19 - [tool monitor]执行工具：rag_summarize
```

格式组成：
- `%(asctime)s`：时间戳
- `%(name)s`：日志器名称
- `%(levelname)s`：日志级别
- `%(filename)s:%(lineno)d`：文件名和行号
- `%(message)s`：日志消息

## 日志在项目中的使用

### 工具调用监控

```python
@wrap_tool_call
def monitor_tool(request, handler):
    logger.info(f"[tool monitor]执行工具：{request.tool_call['name']}")
    logger.info(f"[tool monitor]传入参数：{request.tool_call['args']}")

    try:
        result = handler(request)
        logger.info(f"[tool monitor]工具{request.tool_call['name']}调用成功")
        return result
    except Exception as e:
        logger.error(f"工具{request.tool_call['name']}调用失败，原因：{str(e)}")
        raise e
```

### 模型调用前日志

```python
@before_model
def log_before_model(state, runtime):
    logger.info(f"[log_before_model]即将调用模型，带有{len(state['messages'])}条消息。")
    logger.debug(f"[log_before_model]{type(state['messages'][-1]).__name__} | {state['messages'][-1].content.strip()}")
```

### 知识库加载日志

```python
logger.info(f"[加载知识库]{path}内容已经存在知识库内，跳过")

logger.error(f"[加载知识库]{path}加载失败：{str(e)}", exc_info=True)
```

logger.warning(f"[fetch_external_data]未能检索到用户：{user_id}在{month}的使用记录数据")

## 日志文件管理

日志文件按日期命名，存储在`logs/`目录下：

```
logs/
├── agent_20250125.log
├── agent_20250126.log
└── agent_20250602.log
```

每天一个文件，便于按日期查找问题。

## 防止重复Handler

```python
if logger.handlers:
    return logger
```

这段代码很重要。Streamlit每次交互都会重新执行脚本，如果不做判断，Handler会被重复添加，导致同一条日志输出多次。

## exc_info参数

```python
logger.error(f"[加载知识库]{path}加载失败：{str(e)}", exc_info=True)
```

`exc_info=True`会在日志中记录完整的异常堆栈信息，对于排查问题非常有用：

```
2025-06-02 10:30:15 - agent - ERROR - vector_store.py:98 - [加载知识库]data/broken.pdf加载失败：PDF解析错误
Traceback (most recent call last):
  File "vector_store.py", line 78, in load_document
    documents = get_file_documents(path)
  ...
```

## 总结

日志系统是智能体可观测性的基础。本项目通过双通道输出（控制台INFO + 文件DEBUG）、结构化日志格式、防重复Handler和异常堆栈记录，构建了一个实用且健壮的日志体系。在Agent这种决策过程复杂的系统中，完善的日志记录是开发调试和生产运维的关键保障。
