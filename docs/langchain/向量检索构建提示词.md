# 向量检索构建提示词——RAG的基础实现

RAG（检索增强生成）的核心思想是：先从知识库中检索相关文档，再将检索结果注入提示词，让大语言模型基于参考资料生成回答。本文将从零开始实现RAG的基础流程，展示如何手动完成"查询→检索→注入→生成"的完整过程。

## 核心概念

**RAG核心流程**：
1. **查询（Query）**：用户提出问题
2. **检索（Retrieve）**：从向量存储中检索与问题最相关的文档
3. **注入（Inject）**：将检索到的文档内容作为上下文注入提示词模板
4. **生成（Generate）**：大语言模型基于上下文生成回答

**手动上下文注入**：本文采用手动方式构建上下文——先调用`similarity_search`获取相关文档，再手动拼接为字符串，最后作为变量传入提示词模板。这种方式虽然不够优雅，但能清晰展示RAG的核心原理。

**提示词模板设计**：
- `{context}`变量：接收检索到的参考资料
- `{input}`变量：接收用户的原始问题
- 系统提示要求模型"以已知参考资料为主"回答问题，避免幻觉

**add_texts()方法**：`InMemoryVectorStore`的`add_texts()`方法可以直接添加文本列表，无需先创建`Document`对象，适合快速测试。

## 代码示例

```python
from langchain_community.chat_models import ChatTongyi
from langchain_core.vectorstores import InMemoryVectorStore
from langchain_community.embeddings import DashScopeEmbeddings
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

model = ChatTongyi(model="qwen3-max")
prompt = ChatPromptTemplate.from_messages(
    [
        ("system", "以我提供的已知参考资料为主，简洁和专业的回答用户问题。参考资料:{context}。"),
        ("user", "用户提问：{input}")
    ]
)

vector_store = InMemoryVectorStore(embedding=DashScopeEmbeddings(model="text-embedding-v4"))
vector_store.add_texts(
    ["减肥就是要少吃多练", "在减脂期间吃东西很重要,清淡少油控制卡路里摄入并运动起来", "跑步是很好的运动哦"])

input_text = "怎么减肥？"

result = vector_store.similarity_search(input_text, 2)
reference_text = "["
for doc in result:
    reference_text += doc.page_content
reference_text += "]"

chain = prompt | model | StrOutputParser()

res = chain.invoke({"input": input_text, "context": reference_text})
print(res)
```

## 代码解析

1. **创建提示词模板**：使用`ChatPromptTemplate.from_messages()`定义系统消息和用户消息。系统消息中包含`{context}`占位符用于注入参考资料，用户消息中包含`{input}`占位符用于注入用户问题。

2. **初始化向量存储**：使用`DashScopeEmbeddings`（指定`text-embedding-v4`模型）创建`InMemoryVectorStore`，并通过`add_texts()`直接添加三条文本数据。

3. **检索相关文档**：`similarity_search(input_text, 2)`检索与"怎么减肥？"最相关的2个文档。

4. **手动构建上下文**：遍历检索结果，将每个文档的`page_content`拼接为字符串，用方括号包裹。这是手动注入上下文的关键步骤。

5. **构建链并调用**：`prompt | model | StrOutputParser()`组成处理链，`chain.invoke()`传入包含`input`和`context`的字典，模型基于参考资料生成回答。

## 关键要点

- RAG核心流程：查询→检索→注入上下文→生成回答
- 手动注入上下文：先检索文档，再拼接为字符串，作为`{context}`变量传入模板
- 提示词模板需要`{context}`和`{input}`两个变量分别接收参考资料和用户问题
- `add_texts()`方法可以直接添加文本列表到向量存储
- 系统提示要求模型"以参考资料为主"回答，减少幻觉

## 小结

本文实现了RAG的基础流程，通过手动方式完成了从向量检索到上下文注入的全过程。这种方式虽然代码略显冗长，但清晰地展示了RAG的核心原理：检索相关文档→注入提示词→模型生成回答。在实际项目中，可以使用`RunnablePassthrough`和`as_retriever()`来简化代码，实现更优雅的LCEL风格RAG链。理解手动实现的原理，是掌握高级RAG技术的基础。
