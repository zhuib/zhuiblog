# 文件对话历史存储——自定义BaseChatMessageHistory

在RAG对话系统中，保持多轮对话的上下文连贯性至关重要。LangChain提供了`RunnableWithMessageHistory`来管理对话历史，但它需要一个`BaseChatMessageHistory`的实现来实际存储消息。默认的内存存储在程序重启后丢失，而数据库存储又增加了部署复杂度。文件存储是一个轻量且实用的折中方案——将对话历史以JSON格式写入文件，既持久化又易于查看和调试。本文将介绍如何自定义`FileChatMessageHistory`，实现基于文件的对话历史持久化。

## 核心概念

- **BaseChatMessageHistory**：LangChain定义的对话历史抽象基类，要求实现`add_messages`、`messages`和`clear`三个接口，是`RunnableWithMessageHistory`的存储后端契约。
- **add_messages**：向历史记录中追加新消息的方法，接收`Sequence[BaseMessage]`类型的消息序列。
- **messages属性**：使用`@property`装饰器定义为只读属性，返回当前存储的所有消息列表，类型为`list[BaseMessage]`。
- **message_to_dict/messages_from_dict**：LangChain提供的序列化工具函数，`message_to_dict`将`BaseMessage`转为可JSON序列化的字典，`messages_from_dict`将字典列表还原为`BaseMessage`对象。
- **session_id隔离**：每个会话使用独立的`session_id`作为文件名，实现不同用户/会话的对话历史互不干扰。

## 代码示例

```python
import json
import os
from typing import Sequence
from langchain_core.chat_history import BaseChatMessageHistory
from langchain_core.messages import BaseMessage, message_to_dict, messages_from_dict

def get_history(session_id):
    return FileChatMessageHistory(session_id, "./chat_history")

class FileChatMessageHistory(BaseChatMessageHistory):
    def __init__(self, session_id, storage_path):
        self.session_id = session_id
        self.storage_path = storage_path
        self.file_path = os.path.join(self.storage_path, self.session_id)
        os.makedirs(os.path.dirname(self.file_path), exist_ok=True)

    def add_messages(self, messages: Sequence[BaseMessage]) -> None:
        all_messages = list(self.messages)
        all_messages.extend(messages)
        new_messages = [message_to_dict(message) for message in all_messages]
        with open(self.file_path, "w", encoding="utf-8") as f:
            json.dump(new_messages, f)

    @property
    def messages(self) -> list[BaseMessage]:
        try:
            with open(self.file_path, "r", encoding="utf-8") as f:
                messages_data = json.load(f)
                return messages_from_dict(messages_data)
        except FileNotFoundError:
            return []

    def clear(self) -> None:
        with open(self.file_path, "w", encoding="utf-8") as f:
            json.dump([], f)
```

## 代码解析

**1. 工厂函数**

```python
def get_history(session_id):
    return FileChatMessageHistory(session_id, "./chat_history")
```
`get_history`是`RunnableWithMessageHistory`要求的工厂函数签名，接收`session_id`参数，返回对应的`BaseChatMessageHistory`实例。`RunnableWithMessageHistory`会根据配置中的`session_id`自动调用此函数获取历史记录对象。

**2. 初始化与目录创建**

```python
def __init__(self, session_id, storage_path):
    self.session_id = session_id
    self.storage_path = storage_path
    self.file_path = os.path.join(self.storage_path, self.session_id)
    os.makedirs(os.path.dirname(self.file_path), exist_ok=True)
```
- `session_id`：会话标识，同时作为文件名，不同session_id对应不同文件。
- `storage_path`：存储目录路径，所有对话历史文件统一存放。
- `file_path`：完整文件路径，由目录和session_id拼接而成。
- `os.makedirs`：递归创建目录，`exist_ok=True`表示目录已存在时不报错。

**3. 追加消息**

```python
def add_messages(self, messages: Sequence[BaseMessage]) -> None:
    all_messages = list(self.messages)
    all_messages.extend(messages)
    new_messages = [message_to_dict(message) for message in all_messages]
    with open(self.file_path, "w", encoding="utf-8") as f:
        json.dump(new_messages, f)
```
采用"读取全部→追加新消息→全量写入"的策略：
1. 读取当前所有消息（调用`self.messages`属性）
2. 将新消息追加到列表末尾
3. 使用`message_to_dict`将所有消息序列化为字典
4. 以覆盖模式写入文件

这种策略简单可靠，适合对话历史规模不大的场景。

**4. 读取消息**

```python
@property
def messages(self) -> list[BaseMessage]:
    try:
        with open(self.file_path, "r", encoding="utf-8") as f:
            messages_data = json.load(f)
            return messages_from_dict(messages_data)
    except FileNotFoundError:
        return []
```
使用`@property`将`messages`定义为只读属性，访问时自动从文件加载。`messages_from_dict`将JSON字典列表还原为`BaseMessage`对象列表。首次访问时文件不存在，捕获`FileNotFoundError`返回空列表。

**5. 清空历史**

```python
def clear(self) -> None:
    with open(self.file_path, "w", encoding="utf-8") as f:
        json.dump([], f)
```
清空操作不是删除文件，而是写入空列表`[]`，保持文件存在以便后续追加。

## 关键要点

- `BaseChatMessageHistory`是LangChain对话历史的抽象契约，必须实现`add_messages`、`messages`和`clear`三个接口。
- `message_to_dict`和`messages_from_dict`是LangChain提供的序列化/反序列化工具，确保`BaseMessage`对象与JSON格式之间的正确转换。
- `@property`装饰器将`messages`定义为属性，使得访问方式为`obj.messages`而非`obj.messages()`，符合LangChain基类的接口规范。
- `session_id`作为文件名实现会话隔离，不同用户的对话历史存储在不同文件中，互不影响。
- 文件存储是内存与数据库之间的轻量折中方案，适合开发调试和小规模部署，大规模生产环境建议使用Redis或数据库。

## 小结

本文介绍了如何通过继承`BaseChatMessageHistory`实现基于文件的对话历史存储。`FileChatMessageHistory`利用JSON文件持久化消息，通过`session_id`实现会话隔离，配合`message_to_dict`/`messages_from_dict`完成序列化转换。`get_history`工厂函数是连接`RunnableWithMessageHistory`与自定义存储后端的桥梁。这种方案轻量、直观、易于调试，是RAG项目中对话历史管理的实用选择。
