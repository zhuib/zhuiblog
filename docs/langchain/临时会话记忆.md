# 临时会话记忆——InMemoryChatMessageHistory

大语言模型本身是无状态的，每次调用都是独立的，无法记住之前的对话内容。但在实际应用中，我们常常需要模型具备上下文记忆能力，比如多轮对话场景。LangChain通过`InMemoryChatMessageHistory`和`RunnableWithMessageHistory`提供了临时会话记忆的解决方案。本文将详细介绍如何实现基于内存的会话记忆。

## 核心概念

**InMemoryChatMessageHistory**：基于内存的聊天历史记录器，用于存储单次会话中的消息列表。它的特点是数据存储在内存中，程序重启后数据会丢失，因此适合临时会话场景。

**RunnableWithMessageHistory**：带有会话记忆的Runnable包装器，它能够自动管理消息的存储和读取。每次调用时，它会自动将用户输入和模型输出追加到历史记录中，并在下次调用时将历史消息注入提示词。

**session_id**：会话标识符，用于区分不同的会话。不同的`session_id`对应不同的聊天历史，实现多会话隔离。

**store字典**：用于存储所有会话的历史记录，以`session_id`为键，`InMemoryChatMessageHistory`为值。

**input_messages_key**：指定输入数据中哪个键对应新的用户消息。

**history_messages_key**：指定提示词模板中哪个变量用于接收历史消息。

## 代码示例

```python
from langchain_community.chat_models.tongyi import ChatTongyi
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder
from langchain_core.output_parsers import StrOutputParser
from langchain_core.runnables.history import RunnableWithMessageHistory
from langchain_core.chat_history import InMemoryChatMessageHistory

model = ChatTongyi(model="qwen3-max")
prompt = ChatPromptTemplate.from_messages(
    [
        ("system", "你需要根据会话历史回应用户问题。对话历史："),
        MessagesPlaceholder("chat_history"),
        ("human", "请回答如下问题：{input}")
    ]
)

str_parser = StrOutputParser()

base_chain = prompt | model | str_parser

store = {}

def get_history(session_id):
    if session_id not in store:
        store[session_id] = InMemoryChatMessageHistory()
    return store[session_id]

conversation_chain = RunnableWithMessageHistory(
    base_chain,
    get_history,
    input_messages_key="input",
    history_messages_key="chat_history"
)

if __name__ == '__main__':
    session_config = {
        "configurable": {
            "session_id": "user_001"
        }
    }
    res = conversation_chain.invoke({"input": "总共有几个宠物"}, session_config)
    print("第3次执行：", res)
```

## 代码解析

1. **创建提示词模板**：使用`ChatPromptTemplate.from_messages()`构建消息列表，其中`MessagesPlaceholder("chat_history")`是历史消息的占位符，它会在运行时被替换为实际的聊天历史。

2. **构建基础链**：`prompt | model | str_parser`组成基础的处理链，负责接收输入、调用模型、解析输出。

3. **定义store和get_history**：
   - `store`是一个字典，以`session_id`为键存储各会话的历史记录
   - `get_history`函数根据`session_id`获取或创建对应的`InMemoryChatMessageHistory`

4. **包装为会话链**：`RunnableWithMessageHistory`包装基础链，传入`get_history`函数和键名配置。`input_messages_key="input"`指定输入消息的键，`history_messages_key="chat_history"`指定历史消息的占位符名称。

5. **调用方式**：通过`configurable`配置`session_id`，确保同一会话使用相同的历史记录。每次调用后，用户输入和模型回复会自动保存到对应`session_id`的历史中。

## 关键要点

- `InMemoryChatMessageHistory`将聊天历史存储在内存中，程序重启后数据丢失
- `RunnableWithMessageHistory`自动管理消息的存储和读取，无需手动维护
- `session_id`用于区分不同会话，实现多用户/多会话隔离
- `store`字典以`session_id`为键，管理所有会话的历史记录
- `input_messages_key`指定输入消息的键名，`history_messages_key`指定历史消息的占位符名称
- `MessagesPlaceholder`在模板中为历史消息预留位置

## 小结

`InMemoryChatMessageHistory`提供了最简单的会话记忆实现，适合开发和测试阶段使用。通过`RunnableWithMessageHistory`的包装，我们可以轻松地为任何链添加上下文记忆能力，使模型能够"记住"之前的对话内容。但需要注意，内存存储的局限性在于数据无法持久化，一旦程序重启，所有会话历史都会丢失。对于需要长期保存会话记录的场景，需要使用持久化存储方案（如文件或数据库）。
