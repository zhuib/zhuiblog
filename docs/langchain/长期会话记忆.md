# 长期会话记忆——FileChatMessageHistory持久化存储

`InMemoryChatMessageHistory`虽然简单易用，但数据仅存在于内存中，程序重启即丢失。对于需要长期保存会话记录的应用场景，我们需要将聊天历史持久化到磁盘。本文将介绍如何通过自定义`FileChatMessageHistory`类，实现基于JSON文件的会话记忆持久化存储。

## 核心概念

**BaseChatMessageHistory**：LangChain中聊天历史记录器的抽象基类，定义了所有聊天历史记录器必须实现的接口。自定义持久化存储方案需要继承此类并实现其抽象方法。

**核心方法**：
- `add_messages()`：向历史记录中添加消息，这是写入数据的核心方法
- `messages`：属性方法，返回当前存储的所有消息列表
- `clear()`：清空历史记录

**序列化工具**：
- `message_to_dict()`：将`BaseMessage`对象序列化为字典，便于JSON存储
- `messages_from_dict()`：将字典列表反序列化为`BaseMessage`对象列表，便于从JSON恢复

**文件存储策略**：每个`session_id`对应一个独立的JSON文件，文件名即`session_id`，文件内容为该会话的所有消息序列化结果。

## 代码示例

```python
import os, json
from typing import Sequence
from langchain_core.messages import message_to_dict, messages_from_dict, BaseMessage
from langchain_core.chat_history import BaseChatMessageHistory

class FileChatMessageHistory(BaseChatMessageHistory):
    def __init__(self, session_id, storage_path):
        self.session_id = session_id
        self.storage_path = storage_path
        self.file_path = os.path.join(self.storage_path, self.session_id)
        os.makedirs(os.path.dirname(self.file_path), exist_ok=True)

    def add_messages(self, messages: Sequence[BaseMessage]) -> None:
        all_messages = list(self.messages)
        all_messages.extend(messages)
        new_messages = [message_to_dict(message) for message in all_messages]
        with open(self.file_path, "w", encoding="utf-8") as f:
            json.dump(new_messages, f)

    @property
    def messages(self) -> list[BaseMessage]:
        try:
            with open(self.file_path, "r", encoding="utf-8") as f:
                messages_data = json.load(f)
                return messages_from_dict(messages_data)
        except FileNotFoundError:
            return []

    def clear(self) -> None:
        with open(self.file_path, "w", encoding="utf-8") as f:
            json.dump([], f)
```

## 代码解析

1. **构造函数**：接收`session_id`和`storage_path`两个参数，拼接出完整的文件路径。`os.makedirs()`确保存储目录存在，`exist_ok=True`避免目录已存在时报错。

2. **add_messages()方法**：
   - 先读取当前存储的所有消息（调用`self.messages`属性）
   - 将新消息追加到现有消息列表末尾
   - 使用`message_to_dict()`将所有消息序列化为字典
   - 以覆盖写入模式（"w"）将完整消息列表写入JSON文件

3. **messages属性**：
   - 尝试读取JSON文件并使用`messages_from_dict()`反序列化为`BaseMessage`对象列表
   - 如果文件不存在（首次会话），返回空列表
   - 使用`try/except`处理文件不存在的异常情况

4. **clear()方法**：将空列表写入文件，清空所有历史记录。

5. **使用方式**：与`InMemoryChatMessageHistory`类似，只需在`get_history`函数中返回`FileChatMessageHistory`实例，然后配合`RunnableWithMessageHistory`使用即可。

## 关键要点

- 自定义`FileChatMessageHistory`继承`BaseChatMessageHistory`，实现持久化存储
- 必须实现`add_messages()`、`messages`属性和`clear()`三个核心方法
- `message_to_dict()`将消息对象序列化为字典，`messages_from_dict()`将字典反序列化为消息对象
- 每个会话对应一个独立的JSON文件，以`session_id`为文件名
- `add_messages()`采用"读取全部→追加→写回全部"的策略确保数据完整性

## 小结

通过自定义`FileChatMessageHistory`，我们实现了会话记忆的持久化存储，解决了`InMemoryChatMessageHistory`数据易丢失的问题。这种基于JSON文件的存储方案简单可靠，适合中小规模的应用场景。对于更大规模的生产环境，可以考虑使用数据库（如Redis、MongoDB等）作为存储后端，原理相同，只需替换存储实现即可。持久化会话记忆是实现长期对话型AI应用的基础能力。
